# 多端同步说明

> GitHub 为唯一真相源。Mac/Windows 收集与发布，手机/平板只读。

## 设备分工

| 设备 | 角色 | 做什么 |
|------|------|--------|
| **Mac / Windows** | 收集 + 整理 + 发布 | 收资料、Cursor ingest、git push |
| **手机 / 平板** | 只读 | 复习、查概念、看 Graph（不写 Vault） |

## 目录约定

```
raw/
├── ella/      # Ella 仓库化学资料镜像（更新后 ingest）
├── inbox/     # 待 ingest 的新资料（桌面收集暂存）
└── assets/    # 图片、PDF 附件
wiki/          # 阅读与 Graph 主战场（LLM 维护）
```

## Mac / Windows 工作流

### 首次：在 GitHub 创建空仓库

1. 打开 https://github.com/new
2. 仓库名：**chemistry-wiki**（或与 `origin` 一致的名字）
3. 选 **Private** 或 Public
4. **不要**勾选 "Add a README"（本地已有内容）

### 首次 push（在本 Mac 上）

```bash
cd /Users/hd/python/wiki
./scripts/push-to-github.sh
```

或手动：

```bash
git push -u origin main
```

### 其他电脑首次 clone

```bash
git clone git@github.com:kiss3735707/chemistry-wiki.git
cd chemistry-wiki
```

Obsidian → Open folder as vault → 选择 clone 目录。

### 推荐安装 Obsidian Git 插件

Settings → Community plugins → **Obsidian Git**

建议配置：
- Pull on startup：开启
- Auto commit + push：关闭（手动 commit 更清晰）
- 收集完并 ingest 后，在终端或插件中 **commit + push**

### 日常收集流程

1. 新资料放入 `raw/inbox/` 或从 Ella 同步到 `raw/ella/`
2. Cursor 中说：`ingest raw/inbox/` 或 `ingest raw/ella/ 下变更的文件`
3. 确认 wiki 更新后：

```bash
git add .
git commit -m "ingest: 简述本次更新"
git push
```

### 从 Ella 仓库同步

```bash
# 在 Ella 仓库更新后
cp -r /path/to/Ella-/化学/* raw/ella/
# 然后在 Cursor 中 batch ingest
```

## 手机 / 平板（只读）

### iPhone / iPad

1. 安装 **Working Copy**（Git 客户端）
2. Clone：`git@github.com:kiss3735707/chemistry-wiki.git`
3. Obsidian → Open folder → 选择 Working Copy 中的仓库目录
4. 需要最新内容时：Working Copy → **Pull**（不要在移动端 Push）

### Android 平板 / 手机

1. 安装 Git 客户端（如 MGit）clone 仓库
2. Obsidian 打开该目录
3. 定期 Pull，**不要修改或 Push**

## Graph 阅读建议

Obsidian Graph 设置 Filter：

```
path:wiki
```

优先阅读：[[overviews/初中化学总览]]、[[concepts/]]、[[synthesis/]]

## 注意事项

- 移动端**不要编辑** `wiki/` 下的文件，避免与桌面冲突
- 不要用 iCloud / 网盘直接同步整个 Vault（与 Git 冲突）
- 换电脑前：先 push；换电脑后：先 pull
